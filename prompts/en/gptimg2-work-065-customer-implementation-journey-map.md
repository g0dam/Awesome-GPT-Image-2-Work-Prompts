# Customer implementation journey map

**Target user:** customer success manager

**Use case:** Create an onboarding journey map for a mid-market customer implementation

## Prompt

Create a 16:9 business-ready image for a customer success manager. Goal: Create an onboarding journey map for a mid-market customer implementation. Deliverable: Customer implementation journey map. Visual direction: A customer success playbook map from kickoff, data setup, configuration, training, pilot, go-live and adoption review. Layout and content requirements: show vendor and customer responsibilities in separate lanes; include risk flags, success criteria and meeting cadence; add artifact checklist and milestone markers; use clear swimlanes and concise labels; keep it realistic for onboarding discussions. Use precise layout language, professional work context, crisp readable labels, realistic interface or document details where useful, and enough negative space for website cards and GitHub preview crops. Avoid real company names, vague smiley customer photos, unrealistic promises, clutter, tiny text, decorative-only journey maps.
