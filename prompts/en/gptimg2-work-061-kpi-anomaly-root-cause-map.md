# KPI anomaly root-cause map

**Target user:** operations analyst

**Use case:** Map likely drivers behind a KPI drop for an executive operations review

## Prompt

Create a 16:9 business-ready image for a operations analyst. Goal: Map likely drivers behind a KPI drop for an executive operations review. Deliverable: KPI anomaly root-cause map. Visual direction: A node-link diagnostic board with central KPI drop card and branches for traffic, conversion, fulfillment, support volume and system latency. Layout and content requirements: show linked diagnostic branches with owner chips and confidence tags; include small sparklines and next investigation actions; keep the map readable instead of spaghetti-like; use anomaly colors sparingly; make it executive-ready. Use precise layout language, professional work context, crisp readable labels, realistic interface or document details where useful, and enough negative space for website cards and GitHub preview crops. Avoid fake brand names, excessive numbers, messy lines, alarmist red dashboards, tiny charts, decorative-only infographic elements.
